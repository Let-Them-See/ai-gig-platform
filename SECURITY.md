# Security Policy

## Reporting a Vulnerability

We take the security of GigForge seriously. If you discover a security vulnerability, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

### How to Report

Email us at: **security@gigforge.ai**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fixes

### Response Timeline

- **Acknowledgment**: Within 24 hours
- **Initial Assessment**: Within 72 hours
- **Resolution**: Within 7-14 days (depending on severity)

### Scope

- Authentication and authorization flaws
- Data exposure or leaks
- Payment processing vulnerabilities
- API security issues
- Cross-site scripting (XSS) or injection attacks

### Out of Scope

- Issues in third-party dependencies (report to them directly)
- Social engineering attacks
- Denial of service attacks
- Issues requiring physical access

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| < 1.0   | ❌        |

Thank you for helping keep GigForge and its users safe! 🛡️
